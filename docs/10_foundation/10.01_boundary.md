# 10.01 Boundary

ScienceResearch.jl is a Julia library for research-time algorithm validation.
It gives agents and researchers a disciplined place to test algorithm ideas,
dataset assumptions, feasibility constraints, baseline comparisons, and
notebook-backed evidence before those ideas are promoted into domain projects or
runtime engines.

## Scope

ScienceResearch owns generic research infrastructure:

- experiment descriptors;
- dataset and workload descriptors;
- metric contracts;
- baseline comparison helpers;
- portable result artifacts;
- Pluto notebook validation and static publication helpers.

ScienceResearch does not own domain algorithms. Project-specific work such as
graph search strategy, OCR routing, ontology proof, document extraction, or any
other domain engine belongs in the domain package that needs it. Those packages
may use ScienceResearch to validate and publish evidence.

## Non-Goals

- No project-specific public APIs.
- No runtime gateway ownership.
- No Rust, Python, or Julia engine routing policy.
- No training loop, reinforcement learning framework, or model-serving layer.
- No benchmark result promotion without an explicit experiment artifact.

## Design Rule

If an API name mentions a domain, product, paper, or personal project scenario,
it is probably not a ScienceResearch public API. Keep the public surface about
research contracts and let downstream packages own domain-specific algorithms.
