# 30.01 Pluto Workflow

Pluto notebooks are the review surface for ScienceResearch.jl. They are not the
only research artifact, and they are not the library's core domain. The core
domain is algorithm validation; notebooks make that validation readable and
publishable.

## Notebook Role

A research notebook should:

- state the hypothesis;
- load or construct the dataset and workload descriptors;
- run one or more experiments;
- show metrics and baseline deltas;
- link to generated result artifacts;
- explain whether the idea is worth downstream implementation.

## Notebook Discipline

ScienceResearch validates a small set of notebook hygiene rules:

- a notebook must have a Pluto cell order block;
- code cells should be preceded by markdown context;
- code cells should avoid packing multiple function definitions together;
- generated HTML should keep tables, code, and math readable.

These rules exist to keep algorithm evidence reviewable by humans and agents.

## Publication

`build_notebook_html` and the HTML shell helpers publish notebooks into static
HTML. Publication is an output path. It must not pull domain algorithms into
ScienceResearch.

Before publication, notebooks should reference machine-readable evidence with
`scienceresearch-artifact: <relative-path>`. Run `validate_notebook_evidence`
against the artifact directory so stale or missing experiment manifests are
reported before HTML is generated.
