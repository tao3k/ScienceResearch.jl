# 40.01 Agent Research Loop

ScienceResearch.jl should help an agent test algorithm ideas without confusing
a promising experiment with a production design. The agent loop is deliberately
small and evidence-first.

## Loop

1. State the algorithm idea in an `ExperimentSpec`.
2. State the dataset shape in `DatasetSpec`.
3. State the workload and budget in `WorkloadSpec`.
4. Declare required metrics with `MetricSpec`.
5. Run dataset and algorithm validators before measuring the candidate.
6. Run the candidate and record `ExperimentResult`.
7. Use `benchmark_experiment` when repeated timing samples matter.
8. Compare against a baseline when one exists.
9. Use `decide_research_promotion` to classify the result.
10. Write a result artifact and publish the notebook only after the evidence is
   reproducible.

## Decision Semantics

`ResearchDecision` has a deliberately small vocabulary:

- `:promote`: the candidate satisfies declared thresholds and baseline deltas.
- `:needs_more_evidence`: the candidate failed a threshold, failed a baseline
  delta, or needs another experiment before downstream implementation.
- `:reject`: strict promotion policy rejects failed thresholds or deltas.
- `:stale_evidence`: the evidence artifact is no longer current enough for
  downstream promotion.

ScienceResearch does not decide production ownership. A `:promote` decision
only means the research evidence is strong enough for the owning downstream
package to consider implementation.

## Boundary

AgentAutoResearch can generate candidates, run experiments, compare baselines,
and produce artifacts through ScienceResearch. It must not add domain-specific
algorithm APIs to ScienceResearch. Domain packages should import this library
and keep their own algorithms, fixtures, and promotion gates.

Downstream examples such as graph search, PageRank-style ranking, OCR routing,
or ontology validation should therefore live in their owning packages. They can
use ScienceResearch for benchmark interfaces, dataset validation, algorithm
validation, and notebook publication without turning those examples into
ScienceResearch APIs.
